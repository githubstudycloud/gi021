## 一、背景与目标

本 Skill 面向测试管理平台，提供对**用例树**的结构化查询能力。
核心诉求：将复杂树形结构的多种接口调用封装为可对话的 Skill，让 AI 能自动遍历、增量同步、批量拉取用例详情，并对外提供统一的数据视图。

## 二、树形结构定义

### 2.1 节点类型（type）

| 层级 | 节点类型 | 说明 |
|------|----------|------|
| 第 1 层 | `baseline_version` | 基线版本，树的根（入口） |
| 第 2 层 | `case_dir`（基线目录）、`exec_version`（执行版本） | 基线目录 + 若干执行版本并列 |
| 第 3 层（基线侧） | `case_dir` / `baseline_case` | 用例目录或基线用例，可无限嵌套 |
| 第 3 层（执行版本侧） | `case_dir` 或多个 `scene_dir` | 二选一：要么一个 case 目录，要么多个场景目录 |
| 场景目录下 | `case_dir` → `exec_case` / `case_dir` | 场景目录内部结构同执行版本的 case_dir |

> **约束**：执行版本有场景时，不会同时出现顶层 `case_dir`；二者互斥。

### 2.2 节点 ID 体系

每个节点同时具有两种标识：

| 字段 | 说明 | 示例 |
|------|------|------|
| `uri` / `long_id` | 从根到当前节点的完整路径，以 `/` 分割 | `bv_001/ev_002/sc_003/case_dir/c_004` |
| `short_id` | 当前节点自身的 ID（路径最后一段） | `c_004` |

入口由外部提供一个**默认 `long_id` 路径**，作为遍历起点。

### 2.3 树形结构示意

```
baseline_version (bv_001)
├── case_dir (基线目录)
│   ├── case_dir (子目录, 可嵌套)
│   └── baseline_case (基线用例)
├── exec_version (ev_001)
│   └── case_dir                   ← 无场景时
│       ├── exec_case
│       └── case_dir (子目录)
└── exec_version (ev_002)
    ├── scene_dir (场景A)           ← 有场景时，不会有并列 case_dir
    │   └── case_dir
    │       ├── exec_case
    │       └── case_dir
    └── scene_dir (场景B)
        └── case_dir
            └── exec_case
```

## 四、Skill 设计思路

### 4.1 核心能力

| 能力 | 说明 |
|------|------|
| **树遍历** | 从给定 `long_id` 出发，递归或广度优先遍历整棵树，识别节点类型 |
| **批量拉取** | 收集叶子用例 `uri`，分 200 一批调用详情接口 |
| **增量同步** | 利用 `updated_after` 参数只拉取变更节点；目录移动单独处理 `moved_after` |
| **字段解析** | 调用模板接口，动态理解每个字段含义，辅助 AI 解释数据 |
| **结构感知** | 自动区分"有场景"和"无场景"的执行版本，走不同遍历路径 |

### 4.2 遍历策略

```
输入: default_long_id (baseline_version 的 uri)

1. 查下级节点(default_long_id)
   → 区分 case_dir（基线目录）和 exec_version（执行版本）

2. 对每个 exec_version 查下级
   → 若子节点含 scene_dir → 走"有场景"路径
   → 若子节点含 case_dir  → 走"无场景"路径

3. 递归展开所有 case_dir，收集 baseline_case / exec_case 的 uri

4. 按 200 分批调用批量详情接口

5. （可选）调用全量 URI 接口做完整性校验
```

### 4.3 增量同步策略

```
1. 调用全量 URI 接口获取当前所有 uri
2. 与本地缓存对比 → 新增/删除的节点
3. 调用批量详情接口（带 updated_after）获取变更节点
4. 对目录移动：检测 moved_after，更新路径映射
```
